Git – Version Control System (Detailed Notes)

1. Introduction to Git

Git is a distributed version control system used to track changes in source code during software development. It was created by Linus Torvalds in 2005. Git helps developers collaborate, manage code versions, and maintain project history.

Git is widely used in open-source and enterprise software development.

2. Why Version Control is Important

  • Tracks code changes
  • Enables team collaboration
  • Allows rollback to previous versions
  • Prevents code loss
  • Manages multiple features simultaneously

3. History of Git

  • 2005 – Git developed by Linus Torvalds
  • 2008 – GitHub launched
  • Present – Git is industry standard

4. Features of Git

  • Distributed version control
  • Fast performance
  • Data integrity
  • Branching and merging
  • Offline support

5. Git Architecture

Git consists of the following components:

  • Working Directory
  • Staging Area
  • Local Repository
  • Remote Repository

6. Installing Git

Git can be installed from the official website: git-scm.com. After installation, Git can be accessed via terminal or command prompt.

7. Git Configuration

git config --global user.name "Your Name" git config --global user.email "youremail@example.com"

8. Git Repository

A repository stores all project files and history.

git init

9. Git Status

Shows the current state of the working directory and staging area.

git status

10. Adding Files to Staging Area

git add filename git add .

11. Committing Changes

Commit saves changes to the local repository.

git commit -m "Initial commit"

12. Git Log

Displays commit history.

git log

13. Branches in Git

Branches allow parallel development.

git branch git branch new-feature

14. Switching Branches

git checkout branch-name

15. Merging Branches

Combines changes from different branches.

git merge branch-name

16. Git Remote

Remote repositories store code online.

git remote add origin url

17. Pushing Code to Remote Repository

git push origin main

18. Pulling Code from Remote Repository

git pull origin main

19. Cloning a Repository

git clone repository-url

20. Git Ignore

.gitignore file specifies files Git should ignore.

21. Git Stash

Temporarily saves changes without committing.

git stash git stash pop

22. Git Reset

Used to undo changes.

git reset --hard HEAD

23. Git Rebase

Reapplies commits on top of another base branch.

24. Git vs GitHub

  • Git – Version control system
  • GitHub – Cloud-based Git hosting service

25. Advantages of Git

  • Fast and efficient
  • Supports collaboration
  • Strong branching model

26. Disadvantages of Git

  • Steep learning curve
  • Complex commands for beginners

27. Applications of Git

  • Software development
  • Open-source projects
  • DevOps workflows
  • Team collaboration

28. Summary

Git is an essential tool for modern software development. It helps manage code versions efficiently and supports team collaboration.
